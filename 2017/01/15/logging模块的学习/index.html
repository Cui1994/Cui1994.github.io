logging模块的学习

发表于 2017-01-15 |

这部分内容其实是补充的，用来记录项目进行过程中学习到的logging模块，摘自这里。出发点是为自己项目模块中的三个爬虫部分添加日志记录功能，顺便学习了模块的大概，幸好也没什么难的。

概况

Python的logging模块是通用日志系统，通log4j的机制相似。其中包括几个主要部分：

  • logger：提供日志接口，供程序使用
  • handler：将日志记录发往合适的目的地
  • filter：提供一种优雅的方式决定是否将日志发往目的地
  • formatter：指定日志记录输出的具体格式

具体用法

  • 初始化： logger = logging.getLogger("name"), 这里的name就是要用应用到的模块的名字
  • 设置级别： logger.setLevel(logging.DEBUG)， logging有NOTSET < DEBUG < INFO < WARNING < ERROR < CRITICAL这几种级别，设置后会记录该机别及其以上级别的信息
  • Handler，常用的是StreamHandler和FileHandler， 可以简单理解为一个是console和文件日志，一个打印在shell窗口上，一个记录在一个文件上
  • formatter，定义了最终log信息的顺序,结构和内容。一般用'[%(asctime)s][%(name)s][%(levelname)s] %(message)s', '%Y-%m-%d %H:%M:%S'
